简介

容器与虚拟机

在日常开发中，每个用户计算机的环境都不相同，要保证软件能够顺利跑起来，必须满足两个条件

  1. 操作系统的设置
  2. 各种库和组件的安装

针对上诉问题，如果能在软件安装时，将原始环境复制过来，就可以完美解决。一般有两种解决的办法

  1. 虚拟机
  2. 容器

虚拟机（virtual machine）：带环境安装的一种解决方案。它可以在一种操作系统里面运行另一种操作系统。对于底层系统来说，虚拟机就是一个普通文件，不需要了就删掉，对其他部分毫无影响。但使用虚拟机由以下缺点

  1. 资源占用过多：会占用一部分内存和空间
  2. 冗余步骤多：因为是完全的操作系统，必然会有一些系统级别的操作，如用户登录
  3. 启动慢：毕竟是真正运行一个系统

容器：针对虚拟机的缺点，Linux提供了一种虚拟化技术，Linux 容器（Linux Containers，缩写为 LXC）。Linux 容器不是模拟一个完整的操作系统，而是对进程进行隔离。即底层依赖于Linux，但进程是被隔离的。相比虚拟机，它有如下优点

  1. 启动快：启动容器相当于启动本机的一个进程，而不是启动一个操作系统，速度就快很多。
  2. 资源占用少：只占用需要的资源，且多个容器可以共享资源（虚拟机独享资源）
  3. 体积小：只包含需要的组件（虚拟机时整个操作系统）

此外，还有其他如OpenVZ、Solaris Zones等容器技术。

什么是Docker

Docker是一个能够把开发的应用程序自动部署到容器的开源引擎，能够提供一个轻量且快速的环境，能够运行开发中的程序，并且可以高效地将程序部署到测试环境、生产环境中。

Docker主要提供

  1. 一个简单、轻量的建模方式：只需几分钟即可将程序运用到Dcoker中。
  2. 职责分离：开发者只需关系容器中运行的程序，运维人员只需关心如何管理容器
  3. 快速、高效的开发生命周期

此外，Docker鼓励使用面向服务的架构，即一个容器运行一个应用程序或进程，一次形成一个分布式的应用程序模型。在这种模型下，应用程序或服务都可以表示为一系列内部互联的容器，从而使分布式部署应用程序，扩展或调试应用程序都变得非常简单，同时也提高了程序的内省性

Docker主要功能

Docker的开销比较低，可以用于服务器、台式机或笔记本。它提供了如下功能。

  1. 一个原生的Linux容器格式，Docker中称为libcontainer。
  2. Linxu内核的命名空间（namespace）[9]，用于隔离文件系统、进程和网络。
  3. 文件系统隔离：每个容器都有自己的root文件系统。
  4. 进程隔离：每个容器都运行在自己的进程环境中。
  5. 网络隔离：容器间的虚拟网络接口和IP地址都是分开的。
  6. 资源隔离和分组：使用cgroups[10]（即control group，Linux的内核特性之一）将CPU和内存之类的资源独立分配给每个Docker容器。
  7. 写时复制[11]：文件系统都是通过写时复制创建的，这就意味着文件系统是分层的、快速的，而且
    占用的磁盘空间更小。
  8. 日志：容器产生的STDOUT、STDERR和STDIN这些IO流都会被收集并记入日志，用来进行日志分析
    和故障排错。
  9. 交互式shell：用户可以创建一个伪tty终端，将其连接到STDIN，为容器提供一个交互式的shell

Docker客户端和服务器

Docker是一个客户端/服务器（C/S）架构的程序。由客户端向Dokcer服务器或守护进程发送请求，由服务器或守护进程完成所有工作并返回结果。Docker提供了一个命令行工具docker以及一整套RESTful API[2]来与守护进程交互

Docker守护进程又被称为Dokcer引擎。

Docker镜像

镜像是用于运行容器的基础，它是基于联合（Union）文件系统的一种层式的结构，由一系列指令一步一步构建出来。

Registry

Registry来保存用户构建的镜像，分公有和私有。公有Registry叫做Docker Hub，我们可以在如下网址(https://hub.docker.com/),注册账号保存分享Registry

容器

Docker可以帮用户构建和部署容器，用户只需要把自己的应用程序或服务打包放进容器即可。容器基于镜像启动，且一个容器可以运行一个或多个进程。相当于一个执行环境
